================================================================================
TECHNICAL DETAILS - ROOT CAUSES
================================================================================

WHY COLOR PATCHES OCCUR AT <480px:
- Radial gradients use percentage-based positioning (20%, 80%)
- On 480px viewport: circle at 80% = 384px from top-left
- Gradient radius "50%" = extends 240px from center
- Creates visible colored square in corner
- Multiple overlapping circles create distinct color blocks
- "transparent 50%" creates hard edge, not smooth fade

SOLUTION MECHANICS:
- Add intermediate color stops (30%, 60%) for gradual fade
- Increase transparency radius (60%+) to push visible edge further
- Mobile version uses even lower opacity (0.01-0.02)
- Extra breakpoint for <375px uses solid fallback

WHY WELCOME SCREEN IS PROBLEMATIC:
- Should be neutral before user chooses track
- Currently defaults to IT brown (#1a0f0a)
- Uses IT-theme gradients (orange-based)
- Creates visual bias before user decision
- Violates principle of dual-track design equality
